# 要件定義書

**システム名：** 個別指導塾 授業管理システム  
**作成日：** 2026年3月23日  
**バージョン：** 1.0

---

## 1. プロジェクト概要

### 1.1 背景・目的

個別指導塾において、先生が授業後に提出する報告書をもとに、以下を一元管理するWebアプリケーションを構築する。

- 先生と生徒の組み合わせの管理
- 授業実施回数の集計・管理
- 報告書の提出状況の把握

### 1.2 対象規模

| 対象 | 規模 |
|------|------|
| 先生 | 6〜20名 |
| 生徒 | 31〜100名 |

---

## 2. ユーザー・権限

### 2.1 ユーザー種別

本システムのユーザーは以下の2種類とする。

| ユーザー種別 | 説明 |
|-------------|------|
| 管理者・スタッフ | 塾の運営・管理を行うスタッフ |
| 先生 | 授業を担当する講師 |

### 2.2 権限マトリクス

| 機能 | 管理者 | 先生 |
|------|--------|------|
| 全報告書の閲覧 | ✅ | ❌ |
| 全生徒情報の閲覧 | ✅ | ❌ |
| 全先生・全生徒の授業回数確認 | ✅ | ❌ |
| 未提出アラートの受信・確認 | ✅ | ❌ |
| 先生・生徒のマスタ管理 | ✅ | ❌ |
| 自分の報告書の提出・編集 | ✅ | ✅ |
| 自分が担当する生徒の情報閲覧 | ✅ | ✅ |
| 自分の授業回数の確認 | ✅ | ✅ |

## 3. 機能要件

### 3.1 報告書管理機能

#### 3.1.1 報告書の提出

- 先生がシステム独自のWebフォームから報告書を提出できる
- 報告書の提出項目は以下とする

| 項目 | 必須/任意 | 備考 |
|------|---------|------|
| 授業実施日 | 必須 | |
| 授業時間 | 必須 | |
| 担当先生名 | 必須 | ログインユーザーから自動入力 |
| 生徒名 | 必須 | サジェスト機能あり（後述） |
| 授業内容・進捗メモ | 任意 | |
| 次回の授業予定 | 任意 | |

#### 3.1.2 生徒名サジェスト機能

報告書フォームの生徒名入力欄において、以下の仕様でサジェストを行う。

- 先生名が入力・選択されると、その先生が担当する生徒の名前を優先的にサジェスト候補として表示する
- 担当外の生徒も検索・選択可能とする（サジェスト候補として区別して表示することを推奨）
- 先生と生徒の担当関係は多対多（1人の生徒が複数の先生に習うことがある）
- サジェストのデータソースはGoogleスプレッドシートで管理された先生・生徒マスタを参照する

**Googleスプレッドシート連携方針**

現状、先生と生徒の対応関係はGoogleスプレッドシートで管理されている。担当変更の発生頻度（週1〜月1程度）を踏まえ、**方式B：定期同期** を採用する。

| 項目 | 内容 |
|------|------|
| 連携方式 | 定期同期（Google Sheets API経由） |
| 同期頻度 | 1日1回（時間帯は授業開始前を推奨） |
| 同期内容 | 先生マスタ・生徒マスタ・担当関係 |
| 同期方向 | Googleスプレッドシート → システムDB（一方向） |

- スプレッドシートを更新してから最大1日以内にシステムへ反映される
- 同期失敗時は管理者へアラートを通知する（通知方法は別途検討）
- システム側からスプレッドシートへの書き戻しは行わない

#### 3.1.3 報告書の編集

- 先生は自分が提出した報告書を編集できる
- 管理者はすべての報告書を編集できる

#### 3.1.4 報告書の閲覧

- 先生は自分が提出した報告書のみ閲覧できる
- 管理者はすべての報告書を閲覧できる

---

### 3.2 授業回数管理機能

以下の軸で授業回数を集計・表示する。

| 集計軸 | 閲覧権限 |
|--------|---------|
| 先生ごとの合計授業回数 | 管理者のみ（先生は自分の分のみ） |
| 生徒ごとの合計授業回数 | 管理者のみ（先生は担当生徒の分のみ） |
| 先生×生徒の組み合わせごとの授業回数 | 管理者のみ（先生は自分の分のみ） |
| 月別・期間別の授業回数 | 管理者のみ（先生は自分の分のみ） |

---

### 3.3 通知・アラート機能

| アラート内容 | 対象ユーザー | 条件 |
|-------------|-------------|------|
| 報告書未提出アラート | 管理者 | 授業実施日から一定期間経過しても報告書が提出されていない場合 |

※ アラートの通知方法（メール・システム内通知等）は別途検討する。

---

### 3.4 認証・ログイン機能

- ユーザーはIDとパスワードでログインする
- ログイン後、ユーザー種別（管理者 / 先生）に応じた画面・機能を表示する

## 4. 非機能要件

### 4.1 優先度（高い順）

1. **セキュリティ** — 生徒・先生の個人情報を適切に保護する
2. **早期リリース** — 最低限の機能（MVP）で早期にリリースし、運用しながら改善する
3. **操作性** — 先生・管理者ともに直感的に操作できるUI/UXを提供する
4. **機能充実度** — 将来的に必要な機能を段階的に追加できる拡張性を持たせる

### 4.2 セキュリティ要件

- 通信はHTTPS（SSL/TLS）で暗号化する
- ユーザーは認証なしにシステムにアクセスできない
- 先生は他の先生の報告書・生徒情報にアクセスできない
- パスワードはハッシュ化して保存する
- セッションタイムアウトを設ける

### 4.3 パフォーマンス要件

- 中規模（先生20名・生徒100名）の利用に対して、通常操作のレスポンスは3秒以内とする

### 4.4 可用性

- 平日の授業時間帯（例：14:00〜22:00）を中心に安定稼働すること

## 5. システム構成

| 項目 | 内容 |
|------|------|
| 形態 | Webアプリケーション（ブラウザで利用） |
| 対応デバイス | PC・タブレット（スマートフォン対応は任意） |
| 報告書入力方式 | システム独自のWebフォーム |
| マスタデータ管理 | Googleスプレッドシート（先生・生徒の対応関係） |
| 外部連携 | Google Sheets API（連携方式は要検討） |

### 5.1 データ関係

- 先生と生徒の担当関係は **多対多**（1人の生徒が複数の先生に習うことがある）
- 先生・生徒のマスタデータはGoogleスプレッドシートで管理されており、システムと連携する

## 6. 将来検討事項・未決定事項

### 6.1 スコープ外（将来対応）

以下は今回のスコープ外とし、将来的に検討する。

- データのエクスポート機能（CSV / Excel / PDF）
- 授業回数が一定数を超えた際の通知
- 次回授業予定のリマインダー通知
- スマートフォン対応（レスポンシブ対応）
- 保護者・生徒へのポータル機能

### 6.2 未決定事項（要検討）

| 項目 | 内容 |
|------|------|
| Googleスプレッドシート同期タイミング | 同期実行の時間帯・失敗時の通知方法を決定 |
| 未提出アラートの通知方法 | メール通知 / システム内通知 / その他 |
| データのエクスポート形式 | CSV / Excel / PDF など |
